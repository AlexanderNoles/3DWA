About Page

My website is designed to demonstrate some fundamental shader techniques. This is done through letting users apply various shader effects onto three models that I have created.

3D Models

Spaceship

The first model is based generally on popular notions of flying saucers but also specifcally on one particular example. In the MMO, "Final Fantasy XIV" there is a sequence of comedic questlines which includes an encounter with aliens. They fly the ship pictured below.

Skull

The Skull model draws inspiration from two primary sources, video games "Devil Daggers" and "Waltz of the Wizard". "Devil Daggers" uses a very pixelated and low poly style, one which has been mimicked in my model.
Image Source
Unlike my other models (which use a mix of mostly singular colours and some textures), the Skull uses only textures (made by me in a program called Aseprite), as a result this was by far the hardest model to make. It took longer than both other models combined by a significant margin (around 6 to 8 hours, including the time spent learning how to properly create useful UVs). Visual Effects

The demonstratory effects I've included are listed below, alongside a brief explanation of how I believe they demonstrate shader techniques.
1. Value. Used to demonstrate the fundamental value (or average of all the colours) of a given pixel.
2. Pixelation. Used to demonstrate how you can manipulate UV coords to produce effects. In this case rounding UV coords to a cell center.

Design

I used Bootstrap to handle the layout of my html elements. A canvas, which threejs uses as a render target, is placed behind everything, set to fill the full resolution of the window. Every other element is then placed on top of that. This unintentionally created a usability issue as divs naturally blocked mouse events to the canvas, even if they were invisible. The issue was quickly fixed after it was revealed by user testing, however, any element that needs to handle mouse events now needs to be tagged explicitly.

In general, I utilize a minimalist approach to design. The only colour in the website is used to differentiate and highlight the types of buttons. My "About" link is placed at the bottom of the page, intentionally removed from the design (and placement) of the other buttons.

Integration

Alongside the models, I've also added some audio elements. All the onclick events for every button, also calls a "playButtonClick" function, to add some additional feedback for the user. This is especially important for cases where effects (mostly the "Value" effect on the first two models) don't necessarily have a visible impact.

Interaction

To help with viewing the models, I utilize the "OrbitControls" addon for threejs. This provides a basic orbiting camera, controlled similarly to programs like Blender.

My scene is lit using one directional light and one ambient light, both can have their colour and intensity changed by a user through a GUI. There are buttons to play the animation for a given model, set the current model to wireframe mode and then buttons for toggling a variety of effects. At the very top of the page is a model switching navigation bar.

Testing

Testing involved handing a participant (entirely people from my house) the website, before standing back and simply observing. I explicitly provided no assistance or explanation of the website or the module. This was also typically conducted directly after doing testing for my Final Year Project, I do not believe this had any effect on the results but also believe it is worth noting regardless.
Testing primarily assisted in gauging the usability of the website (the previously discussed audio feedback additions were a result of this) but also had an effect on it's design. Originally I decided on a less minimalist design approach but a common point of criticism I recieved was the lack of focus on the models. I ended up focusing on removing visual clutter, getting rid of some unnecesarry explanation text and indents on the buttons.
